Tokyo Trip · 導覽原型

三種找到旅程資訊的方法

請先用手機瀏覽，再以「第一次使用時，哪一套最不需要解釋」作為選擇標準。

A

需求問答式

先問使用者現在想做什麼，再帶往對應工具。

  • 第一眼只有四個主要選擇
  • 適合不熟悉網站功能的家人
  • 內頁有清楚的下一步
開啟 A 方案
B

分類儀表板式

按使用情境分區，同時保留較高的功能可見度。

  • 功能容易瀏覽與比較
  • 適合日後增加更多工具
  • 桌面版資訊密度較高
開啟 B 方案
C

旅程階段式

根據出發前、旅行途中和需要協助切換工具。

  • 最貼近實際旅行情境
  • 途中常用功能集中
  • 具有清楚的階段感
開啟 C 方案
D

混合導覽式

入口先選需求，進入分類後再依旅程階段細分。

  • 首頁選項少而明確
  • 分類內保留旅行情境
  • 結合 A 與 C 的優點
