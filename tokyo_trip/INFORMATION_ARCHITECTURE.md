# 東京旅程中心｜正式資訊架構

## 導覽原則

入口首頁先回答「現在想找什麼？」；進入分類後，依該類資訊的自然結構分類，而不是強迫使用同一組旅程階段。`trip_timeline.html` 歸在「做好準備 → 行程與旅費」，避免首頁與全站主導覽出現太多同等重要的按鈕。

## 入口層級

```text
首頁 index.html
├─ 查看交通 transportation.html
│  ├─ 全程路線：transportation_guide / 票券與費用比較
│  ├─ 河口湖・箱根：fuji_hakone_rail / 當地交通與周遊券
│  └─ 東京・成田：tokyo_rail / 成田返程
├─ 尋找景點 attractions.html
│  ├─ 河口湖：景點導覽 / 當地交通
│  ├─ 箱根：景點導覽 / 當地交通
│  └─ 東京：景點導覽 / 市區鐵路
├─ 做好準備 preparation.html
│  ├─ 行前清單：packing_checklist / 交通確認
│  ├─ 行程與旅費：trip_timeline / budget
│  └─ 旅途實用：japanese_phrases / search
└─ 其他工具 tools.html
   ├─ 日語溝通：japanese_phrases
   ├─ 旅費管理：budget
   └─ 搜尋與收藏：search
```

## 全站快速入口

- 首頁：回到需求入口。
- 準備：進入「行程與旅費」後開啟可編輯時間軸。
- 搜尋：跨頁搜尋與收藏。
- 緊急協助：在所有既有內容頁保持明顯入口。

## 舊頁面保留策略

既有內容和互動功能不刪除，先以分類頁作為新的入口層。交通總覽、區域交通、東京鐵路、景點導覽、行前清單、日語、旅費、搜尋與緊急手冊都保留原始網址，降低既有書籤與外部連結的風險。

## 命名規則

- 首頁使用使用者語言：查看交通、尋找景點、做好準備、其他工具。
- 分類頁使用情境語言：出發前、旅行途中、需要協助。
- 具體頁面保留可搜尋的內容名稱：交通完全攻略、河口湖・箱根交通、東京鐵路導覽、行前準備清單等。
- 不使用「入口」「移動」「工具總覽」作為使用者主要按鈕名稱。
